Lab 1, CSC/CPE 203 - Python to Java (types)

Orientation – Explicit Types, semi-colons, and curly braces

This lab is designed to highlight some of the differences between Python and Java. Many of the differences are purely syntactical. Other features, like for loops, function differently. Those already familiar with Java should finish the lab and then offer to help others.

Objectives

  • To submit an assignment using github.
  • To translate a Python program to Java.
  • To develop and run a complete Java program.
  • To compile and run a Java program on unix1-5

Resources

  • Your peers
  • Your instructor
  • The Java Standard Library - Bookmark this web page, you will be using the Java Standard Library documentation regularly this quarter.

Administration

Before you get started, you need to set things up. Assuming you've created your git repositories as requested, you next need to copy this lab into your turnin directory. Copy the directory lab01 to the turnin-XXX directory you created when you accepted the "turn in" assignment, where "XXX" is your github ID. You'll do your work in this copy. In Unix, you can copy a file or directory using the cp command. For example, from the main directory for CSC 203, you can copy this lab into your turnin directory with: 

cp -r materials/labs/lab01 turnin-XXX/
where XXX is your github ID.

You need to create two files in your turnin-xxx directory: email.txt and name.txt In email.txt, put your @calpoly.edu e-mail address. In name.txt, put your name, exactly as it appears in PolyLearn.

Be sure to read all of the directions in materials/setup.txt. In particular, make sure you create a repository called "turnin-XXX" (where XXX is your github login ID), by accepting the assignment link given in the first bullet point in materials/setup.txt. This step is necessary for you to receive grades.

Completing the above steps will be part of your grade.

Ground Rules

You may collaborate with your peers as much as you like on this lab. Labs are for your practice. Make sure you can do every lab!

Setup

For this lab you are required to compile and run Java from the command-line.

To compile the java lab code file, type:

javac Lab01.java

To run the code then type:

java Lab01

 

Specification

 
  1. The starting code is in a directory called src. Throughout this class, we will always put source code in a directory called src.
  2. From the starting code run lab01.py. Make sure you understand how everything in this file works. If you need help (or haven’t learned Python) please ask me or a classmate to explain things. You will NOT be tested on Python this quarter, but will need to understand this file enough to write an equivalent java version.  
  3. Now take a look at the file Lab01.java. This file is incomplete and you will need to finish it. Translate the given Python file to Java. Please translate everything, including the comments!   To compile your files, type:

    javac Lab01.java

    To run the code then type:

    java Lab01

Submitting the Lab

Commit your working program to Github. See info/git_commands.txt for details on the commands git add, git commit and git push that you use to do this. By the day after the due date, be prepared to demonstrate your working program to your instructor, and show your source code.

If you'd like to submit the lab one day late with a penalty of 5%, please add a file called late_5_percent_off.txt to the lab01 directory. If you'd like to submit the lab two days late with a penalty of 15%, add a file called late_15_percent_off.txt to the lab01 directory. Make sure you add and push the late file.

After your code is working and submitted, check your submission using the checkgit command on the unix server. If the autograder can't compile your lab for a reason that is caught by checkgit, you will get a zero on the lab. To check your submission, on a Unix server type: 

checkgit <git-login> 203 lab01

If it works, you will see the message "** Tests Passed **". If not, you will see some kind of error message. The error message might not tell you exactly what you need to change, but it should give an indication as to what is wrong.
